Execution graph dynamics for current Task-World activities

This deck pressure-tests the proposed model: one edge concept, typed ports, controller consumed records, leases, snapshots, and graph patches. Each activity shows the node shape and the graph before and after the controller accepts the outcome.

The primitive: node records and input bindings

Instead of first-class data/control/policy edge types, nodes expose typed output ports. Edges are input bindings. The scheduler and controller care about concrete readiness, authority, and resource rules.

Output data

Typed JSON, findings, summaries, checklist deltas, evidence, test reports, clarification answers.

File state

Git commit or worktree snapshot plus an outside-Git manifest for declared external and ignored files.

Graph records

Outcome reports, appeals, graph patch requests, lease events, and scheduling facts consumed by the controller.

A node can emit all three layers. Future nodes receive data and file-state through input bindings. The graph controller consumes outcome and patch records, validates authority, and changes the graph.

What the pressure test suggests

Keep as hard restrictions

  • Past attempt nodes are never rewritten from complete to incomplete.
  • All agent callbacks require lease, attempt, and snapshot identity.
  • Only the graph controller applies graph actions.
  • Only planner/oversight roles can request graph topology changes.
  • Write nodes require an exclusive write lease; read nodes run on stable snapshots.

Leave as policy choices

  • Whether readers may run while a writer is active.
  • Whether a blocked worker session is suspended or terminated.
  • Whether ignored files are captured, forbidden, or classified as ephemeral.
  • Whether review workbench actions are in the main graph or a post-run graph.
  • Whether planner patches are applied immediately or queued for approval.
The model stays understandable if every activity reduces to: node emits typed records; controller validates authority and resource constraints; graph changes by adding new nodes, edges, leases, artifacts, and snapshots rather than mutating historical facts.
