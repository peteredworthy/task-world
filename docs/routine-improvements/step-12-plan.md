# Step 12: Context summarization with critical-aspect preservation (A13)

## Milestone
M4: Schema & Architecture Extensions

## Purpose
Enable routine authors to opt into context summarization for large artifacts (plan, architecture docs). Summaries are generated by a configurable (cheap by default) model, with a safety mechanism that verifies critical aspects are preserved.

## Prerequisites / Dependencies
- Step 5/6 (prompt changes in M2) — this step adds summarization to the prompt builder, which should be stable first.
- The `context_from` schema in `models.py` is the extension point.

## Functional Contract

### Inputs
- `ContextFromConfig` with `summarize: true` and optional `critical: "description"`
- Artifact content (e.g., plan.md, architecture.md)
- Summarization model (configurable, default: cheap model like Haiku)

### Outputs
- **Summarized context:** Compact summary injected into downstream task prompts
- **Cache hit:** Previously generated summary returned (keyed by artifact path + content hash)
- **Critical check pass:** Summary contains critical aspects -> used as-is
- **Critical check fail:** Re-summarize with explicit preservation instruction (max 2 iterations)

### Errors
- Summarization model failure -> fall back to full artifact content (no summary)
- After 2 failed critical-aspect iterations -> use last summary with warning

### Schema Addition
```python
class ContextFromConfig(BaseModel):
    summarize: bool = False
    critical: str | None = None
    summarize_model: str | None = None
```

### Cache Design
- Key: `(artifact_path, content_hash)`
- Lifetime: run duration
- Location: `src/orchestrator/workflow/summary_cache.py`

## Files Modified
- `src/orchestrator/config/models.py` — extend `ContextFromConfig`
- `src/orchestrator/workflow/prompts.py` — integrate summarization into prompt assembly
- `src/orchestrator/workflow/summary_cache.py` — new file for summary caching

## Verification Strategy
- **Unit test:** Summary generation produces shorter output than original.
- **Unit test:** Cache hit returns cached summary without calling model.
- **Unit test:** Missing critical aspect triggers re-summarization (up to 2 iterations).
- **Unit test:** Model failure falls back to full content.
- **Integration test:** End-to-end prompt assembly with summarized context.
- **Regression:** Existing prompt and context_from tests pass.
