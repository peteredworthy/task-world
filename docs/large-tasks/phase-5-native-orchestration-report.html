Evidence

Standard bundles are first-class API output

GET /api/runs/{run_id}/evidence scans the run worktree for validated phase4.evidence.v1 bundles and returns typed planner-facing data. This lets the next planner consume evidence without reconstructing it from logs or ad hoc files.

Iteration findings

Remaining scope

What phase 5 v1 deliberately does not include

This implements native orchestration primitives, not a full visual oversight-tree UI. Review artifacts are exposed first as structured evidence; a dedicated persisted review-artifact table can be added later if the workflow needs richer querying.
