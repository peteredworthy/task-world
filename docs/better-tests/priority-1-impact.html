Test Strategy ∙ Priority 1 ∙ Implementation Report

Enum Codegen

A single Python script that reads all backend enums at commit time and writes the TypeScript types — turning enum drift from a silent runtime mystery into a compile-time error.

Python → TypeScript 12 enums · 54 members ~100ms pre-commit check 3 bugs caught 5 drift classes eliminated

What made this invisible:

  • → TaskStatus was typed as string in many callers — missing values passed silently
  • → cancelled in a union is never flagged "extra" by TypeScript
  • → AgentRunnerType was type = string — any invalid string accepted
  • → No CI check compared Python ↔ TypeScript at any point

06 Classes of Issues Now Prevented

01
Missing frontend type
Python adds a new enum member → the generated TS file is instantly stale → pre-commit blocks the commit → developer regenerates → TypeScript union now includes the new value. A missing type can never silently reach production.
02
Phantom frontend type
A value that exists in TS but never in Python (e.g. 'cancelled' in RunStatus). The generated file is authoritative and only contains values from the Python source. There is no mechanism by which a phantom value can enter the generated file.
03
Incomplete switch exhaustion
When the union is strict, TypeScript's exhaustiveness checking flags any switch that doesn't handle the new member. Bug 3 (taskStatusColor) was discovered exactly this way — the moment the loose string type became a tight union, the compiler flagged the gap.
04
Over-permissive types masking invalid values
AgentRunnerType = string accepted any string. Now it's a union of exactly 7 values that mirror the Python enum. A new agent runner type added to only one side fails to compile on the other.
05
Manual sync errors (general)
Previously, keeping Python and TypeScript in sync required a human to remember to update a separate file. The codegen eliminates the human step entirely — the generated file is the only TS enum surface, and it is always derived from Python.

07 Implementation

scripts/export_enums.py — core logic
def is_str_enum(obj) -> bool:
    """Return True if obj is a class that
    inherits from both str and Enum."""
    return (
        inspect.isclass(obj)
        and issubclass(obj, str)
        and issubclass(obj, Enum)
        and obj is not Enum
    )

def enum_to_ts_union(enum_cls) -> str:
    """Convert a str-enum to a TS union type."""
    members = [f"'{m.value}'" for m in enum_cls]
    union = " | ".join(members)
    return f"export type {enum_cls.__name__} = {union};"

def generate_output(module) -> str:
    lines = [HEADER.format(source=SOURCE_REL)]
    # __dict__ preserves source-file order
    for _name, obj in module.__dict__.items():
        if is_str_enum(obj):
            lines.append(enum_to_ts_union(obj))
            lines.append("")
    return "\n".join(lines)
