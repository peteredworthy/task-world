Discoveries

What we found during implementation

Priority 3 was 70% pre-implemented The earlier "Fix 7 state/UI bugs" commit had already created useWebSocket.test.tsx with processEvent tests. The gaps were: count guards (no extra invalidations), and explicit tests for checklist_gate_evaluated / grades_evaluated.
clarification_responded has a silent no-op path Without payload.task_id, only the activity feed is invalidated — nothing else updates. This was undocumented and untested. Now enforced by the count guard.
autoOpenedRef fix was already correct The ref stores a task_id:action_type key (not a boolean), so a second different action correctly auto-opens. Tests confirm the expected behaviour is stable.
page.routeWebSocket() is the right tool (Playwright 1.48+) The strategy doc suggested page.evaluate() monkey-patching. The native API is cleaner, more reliable, and supports ws.send() directly without touching the browser global.
LIFO route ordering enables dynamic test state Registering a second page.route() for the same URL pattern after setupRoutes() overrides it. Closing over a mutable variable makes response data dynamic mid-test — no "update route" API needed.

Protection

Classes of issue now protected against

P3 — Unit
Cache invalidation drift
Extra invalidateQueries call added silently → over-fetching. Count guards catch this on the next vitest run.
P3 — Unit
Missing event branch
Refactor accidentally drops checklist_gate_evaluated from its branch. Explicit tests fail immediately.
P4 — E2E
Pause banner regression
Renaming a pause_reason value silently shows Paused (old_name). All 24 banner tests fail if any message changes.
P4 — E2E
Status badge / button logic
Pause button appears on a stopping run. The stopping test explicitly asserts all three buttons are absent.
P4 — E2E
isRunStuck() render path
Previously zero coverage (strategy doc bug #7). Stuck-run test loads a failed-at-max-attempts task and asserts the banner renders.
P4 — E2E
WS event doesn't reach UI
processEvent removes approval_requested invalidation. WS injection test verifies the full stack: WS frame → cache → refetch → banner within 3s.
P4 — E2E
autoOpenedRef regression
Ref changes from key-tracking to boolean → only first action ever auto-opens. Two tests catch this: on-load + dismiss-then-second.
