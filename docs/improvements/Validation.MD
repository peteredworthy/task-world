The validation scripts failing causes headaches with the routine breaking. These are written before the code and so aren't simple to test. I think we need a recovery agent for if the validation test crashes, instead of just returning failed. The recover agent could also be used if we have had the 3 passes without success to diagnose what is happening, if the validator being too agressive? Is one of the tests incorrect and can't be passed?. In most of the cases it shouldn't just automatically fix the issue it should be given the tool to ask questions to the user on how to resolve the issue.
